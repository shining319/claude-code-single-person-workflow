# Product Manager / 产品经理

[English](#english) | [中文](#中文)

## English

### Overview

Senior product manager for requirements analysis, user research, and PRD creation. Transform ideas into structured product requirements with user personas and comprehensive documentation.

### Features

- **Requirements Analysis**: Deep understanding of business goals and user needs
- **User Research**: Detailed user personas and scenario mapping
- **Feature Planning**: P0/P1/P2 prioritization and MVP definition
- **PRD Creation**: Comprehensive product requirements documents
- **Risk Assessment**: Feasibility analysis and risk identification

### Installation

```bash
claude plugin install github:shining319/claude-code-single-person-workflow/plugins/product-manager
```

### Usage

Use this plugin when you need to:
- Define product requirements
- Create user personas
- Plan MVP features
- Write comprehensive PRDs
- Assess product feasibility

### Deliverables

- Product Requirements Document (PRD)
- User Personas
- User Scenarios
- Feature Priority Matrix
- Risk Assessment

### Output File Locations

All product documents are saved to `outputs/<project-name>/docs/`:

```
outputs/
└── <project-name>/
    └── docs/
        ├── prd.md
        ├── user-personas.md
        ├── feature-specs.md
        └── mvp-plan.md
```

**File Naming Convention:**
- Use kebab-case: `task-management-prd.md`
- Include version/date when needed: `prd-v1.0.md`
- Use descriptive names: `e-commerce-feature-specs.md`

**Alternative:** Traditional project structure using `./docs/` directory.

## 中文

### 概述

资深产品经理，负责需求分析、用户研究和PRD创建。将想法转化为结构化的产品需求，包含用户画像和全面的文档。

### 功能特性

- **需求分析**: 深入理解业务目标和用户需求
- **用户研究**: 详细的用户画像和场景映射
- **功能规划**: P0/P1/P2优先级和MVP定义
- **PRD创建**: 全面的产品需求文档
- **风险评估**: 可行性分析和风险识别

### 安装

```bash
claude plugin install github:shining319/claude-code-single-person-workflow/plugins/product-manager
```

### 使用场景

当您需要以下帮助时使用此插件：
- 定义产品需求
- 创建用户画像
- 规划MVP功能
- 编写全面的PRD
- 评估产品可行性

### 交付物

- 产品需求文档（PRD）
- 用户画像
- 用户场景
- 功能优先级矩阵
- 风险评估

### 文件输出位置

所有产品文档保存到 `outputs/<project-name>/docs/` 目录：

```
outputs/
└── <project-name>/
    └── docs/
```
