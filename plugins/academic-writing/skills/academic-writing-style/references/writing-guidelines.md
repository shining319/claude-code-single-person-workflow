# 写作指导原则

## 核心原则

### 1. 自然流畅的叙述
避免机械的结构化标记（如"首先...其次...最后"），让内容自然过渡。段落之间通过话题延伸或章节标题来引导，而非明显的转折词。

### 2. 个人化但不过度
- 可以使用第一人称（"笔者"、"我"、"from my experience"等），但不要频繁使用
- 技术分析类作业中个人化程度应适当降低，保持相对客观
- 通过具体观察、实例和细节来体现个性，而非单纯的观点表达

### 3. 章节划分清晰
- 使用描述性的章节标题（如"从繁琐到简洁：Spring Boot如何改变Java开发"）
- 标题应概括内容，而非泛泛的"引言"、"分析"、"结论"
- 每个章节内部内容连贯，避免突兀的列表

### 4. 避免过度格式化
- 尽量减少项目符号和有序列表的使用
- 将信息融入段落叙述中，使用自然语言表达（如"关键因素包括性能、可维护性和扩展性"）
- 只在确实需要列举并列要点时才使用列表

### 5. 句式多样但不复杂
- 混合使用长句和短句，避免单调
- 英文写作：使用常见学术词汇，避免生僻高级词
- 英文写作：避免过度复杂的从句嵌套，保持雅思6分左右的难度
- 中文写作：使用地道的中文表达，避免英文直译的句式

## 具体技巧

### 段落衔接方式

**方式一：话题延伸**
前一段末尾引出的概念，在下一段开头继续深化：

```
...这种自动配置机制大幅降低了开发门槛。

自动配置的实现依赖于Spring的条件注解系统...
```

**方式二：对比转折**
不使用"然而"、"但是"等明显的转折词，而是通过内容自然呈现对比：

```
...Netflix的实践展示了微服务的优势。

这种架构模式也带来了新的挑战...
```

**方式三：问题引导**
通过隐含的问题意识推进内容：

```
...系统性能问题日益突出。

从监控数据来看，最突出的瓶颈在数据库层面...
```

### 避免AI痕迹的表达替换

**避免使用** | **建议替换**
--- | ---
首先...其次...最后 | 按话题自然展开，或使用时间顺序、逻辑顺序
总的来说、总而言之 | 省略，或用"从目前的情况看"、"综合这些因素"
如上所述、如前所述 | 直接重述要点或省略
该技术、本文、上述方法 | 使用具体名称（如"Docker"、"这种优化方案"）
具有以下特点： | 融入段落（如"Docker的轻量级特性使其..."）

### 个人化表达的使用时机

**适合使用第一人称的场景：**
- 描述实际经验："笔者在项目中遇到过..."、"from my experience working with..."
- 表达个人观点："我认为这个设计存在..."、"I found that..."
- 案例分析中的反思："如果重新设计，我会..."、"looking back, I would..."

**应保持客观的场景：**
- 技术原理解释："Docker使用Linux内核特性..."
- 文献综述："Smith等人的研究表明..."
- 纯技术分析："该算法的时间复杂度为O(n)..."

### 例子和细节的运用

**具体化描述：**
- 避免："系统性能显著提升"
- 改为："页面加载时间从8秒降到2秒，数据库查询压力下降60%"

**使用真实案例：**
- 避免："许多大公司都在使用微服务"
- 改为："Netflix将单体应用拆分为数百个微服务，整个过程花费数年时间"

**提供背景细节：**
- 避免："数据库性能不佳"
- 改为："商品详情页的查询涉及7个表的关联，产生了大量N+1查询问题"

## 中英文特殊注意事项

### 中文写作
- 正确使用中文标点：逗号（，）、句号（。）、冒号（：）、引号（""）
- 避免英文标点混用，如逗号(,)、句号(.)
- 专业术语可适当保留英文，但不要过多："Spring Boot框架"、"Redis缓存"
- 数字和单位之间保持一致性："5万笔订单"或"50,000笔订单"

### 英文写作
- 使用正确的英文标点：逗号(,)、句号(.)、冒号(:)、引号("")
- 雅思6分词汇水平：prioritize用prefer, demonstrate用show, subsequently用then/later
- 避免过长的复合句，一句话不超过30词为宜
- 缩写首次出现时应全拼：Object-Relational Mapping (ORM)
- 段落长度适中，每段3-6句话

## 质量检查清单

写作完成后，检查以下要点：

- [ ] 是否避免了"首先...其次...最后"的结构？
- [ ] 是否减少了项目符号和有序列表？
- [ ] 段落之间是否自然衔接？
- [ ] 是否使用了具体的例子和细节？
- [ ] 章节标题是否描述性强且清晰？
- [ ] 第一人称使用是否适度？
- [ ] 标点符号是否符合目标语言规范？
- [ ] 句式是否有长短变化？
- [ ] 语言是否避免了明显的AI痕迹？
- [ ] 技术术语使用是否准确且一致？
