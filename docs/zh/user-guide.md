# 用户指南

## 概述

本指南将帮助您有效使用单人工作流插件，简化您的产品开发流程。无论您是构建完整产品还是处理特定组件，这些插件都能提供专业化的帮助。

## 快速参考

| 任务 | 技能/代理 | 自然语言触发 | 斜杠命令 |
|------|-----------|-------------|---------|
| 数据库设计 | 数据库设计器 | "为...设计数据库" | `/database-designer` 或 `/spw-db` |
| 产品需求 | 产品经理 | "为...创建PRD" | `/product-manager` 或 `/spw-prd` |
| UI/UX设计 | UI设计师 | "为...设计UI" | `/ui-designer` 或 `/spw-ui` |
| 技术栈选择 | 解决方案架构师 | "帮我选择技术栈..." | `/solution-architect` 或 `/spw-arch` |
| 学术写作 | 学术写作 | "帮我写研究报告..." | `/academic-writing` 或 `/spw-writing` |
| 完整产品 | 全栈产品构建者 | "我想构建一个完整产品..." | `/build-dev-workflow` |

## 使用技能:两种方法

本市场中的所有技能都可以通过两种方式激活:

### 方法一:自然语言(自动)
只需用自然语言描述您想要的内容。Claude Code 会根据您的请求自动检测并激活相应的技能。

**示例:**
```
"为电商系统设计数据库"
→ 自动激活数据库设计器技能
```

### 方法二:斜杠命令(显式)
使用斜杠命令直接、显式地激活技能。这种方式更快速,并能保证激活您想要的确切技能。

**可用的斜杠命令:**

#### 单技能插件命令
- `/academic-writing [作业类型和主题]` - 学术写作助手
- `/database-designer [业务领域或需求]` - 完整数据库架构设计
- `/product-manager [产品想法或需求]` - 产品管理和PRD创建
- `/ui-designer [要设计的页面或功能]` - UI/UX设计与规范
- `/solution-architect [系统或应用需求]` - 技术架构设计

#### 套件插件命令 (spw = 单人工作流)
如果您安装了 product-development-suite 或完整市场:
- `/spw-db [需求]` - 数据库架构设计
- `/spw-ui [页面或功能]` - UI/UX界面设计
- `/spw-prd [产品想法]` - 产品管理和PRD
- `/spw-arch [系统需求]` - 技术架构
- `/spw-writing [作业主题]` - 学术写作

#### 工作流命令
- `/build-dev-workflow [产品描述]` - 完整产品开发工作流

**示例:**
```
/database-designer "设计一个包含用户、文章、评论的博客系统数据库"
```

## 独立技能

### 1. 学术写作风格

**用途:** 为中英文大学作业提供专业学术写作。

**何时使用:**
- 撰写研究论文
- 创建技术报告
- 准备项目文档
- 起草案例研究和分析

**斜杠命令:**
- `/academic-writing [作业类型和主题]`
- `/spw-writing [作业类型和主题]` (如果使用套件)

**自然语言触发:**
```
"帮我写一篇关于云计算的技术报告"
"我需要创建一篇关于AI伦理的研究论文"
"撰写关于微服务架构的学术分析"
```

**斜杠命令示例:**
```
/academic-writing "关于云计算安全的技术报告"
/spw-writing "关于医疗AI伦理的研究论文"
```

**输出内容：**
- 自然流畅的文字
- 恰当的学术结构
- 清晰的章节划分
- 专业且易懂的语言

### 2. 数据库设计器

**用途:** 完整的数据库架构设计,包含ER图和SQL脚本。

**何时使用:**
- 设计新数据库架构
- 为应用程序创建数据模型
- 生成ER图
- 编写迁移脚本

**斜杠命令:**
- `/database-designer [业务领域或需求]`
- `/spw-db [需求]` (如果使用套件)

**自然语言触发:**
```
"为电商系统设计数据库"
"为项目管理工具创建架构"
"我需要为社交媒体平台设计数据库"
```

**斜杠命令示例:**
```
/database-designer "包含商品、用户、订单、支付的电商系统"
/spw-db "包含任务、团队、里程碑的项目管理工具"
```

**输出内容：**
- 详细的Markdown设计文档
- 可执行的SQL脚本（MySQL、PostgreSQL、SQL Server）
- DrawDB兼容的JSON/DBML文件
- ER图规范

**特性：**
- 逻辑关系（不包含物理外键）
- 合理的字段大小
- 优化的索引
- 所有对象的强制性注释

### 3. 产品经理

**用途:** 将想法转化为专业的产品需求文档(PRD)。

**何时使用:**
- 分析产品需求
- 创建用户画像
- 编写PRD
- 规划MVP功能
- 定义用户故事

**斜杠命令:**
- `/product-manager [产品想法或需求]`
- `/spw-prd [产品想法]` (如果使用套件)

**自然语言触发:**
```
"为移动任务管理应用创建PRD"
"分析健身追踪平台的需求"
"帮我为在线教育系统定义用户画像"
```

**斜杠命令示例:**
```
/product-manager "具有团队协作功能的移动任务管理应用"
/spw-prd "具有社交功能的健身追踪平台"
```

**输出内容：**
- 全面的需求分析
- 用户画像和场景
- 功能规格说明
- PRD文档
- 可行性评估

### 4. UI设计师

**用途:** 将产品需求转化为专业的UI/UX设计规范。

**何时使用:**
- 规划页面结构和信息架构


- 基础设施规划
- 架构审查报告

## 工作流代理

### 1. 全栈产品构建者

**用途:** 编排从概念到部署的整个产品开发过程。

**何时使用:**
- 从零开始构建完整产品
- 需要端到端工作流协调
- 希望自动任务编排

**斜杠命令:**
- `/build-dev-workflow [产品描述]`

**自然语言触发:**
```
"我想构建一个完整的电商平台"
```

**斜杠命令示例:**
```
/build-dev-workflow "具有商品目录、购物车和支付处理的电商平台"
```

**工作流:**
1. 需求收集 → 产品经理
2. 数据库设计 → 数据库设计器
3. UI/UX设计 → UI设计师
4. 技术架构 → 解决方案架构师
5. 文档编写 → 学术写作
6. 实施协调

### 2. 产品需求分析师

**用途：** 深度分析产品需求和可行性。

**何时使用：**
- 评估新产品想法
- 分析市场适配性
- 评估技术可行性

### 3. 数据库架构规划师

**用途：** 全面的数据库规划和优化。

**何时使用：**
- 复杂数据库设计项目
- 需要性能优化
- 考虑扩展性

### 4. UI/UX设计协调员

**用途：** 协调完整的UI/UX设计流程。

**何时使用：**
- 大规模设计项目
- 多页面/组件设计
- 创建设计系统

### 5. 技术解决方案设计师

**用途：** 高级技术架构和系统设计。

**何时使用：**
- 复杂系统架构
- 微服务设计
- 云原生应用

### 6. 文档生成器

**用途：** 自动生成全面的项目文档。

**何时使用：**
- 创建技术文档
- API文档
- 用户手册和指南

## 最佳实践

### 1. 具体明确

不要使用：
```
"设计一个数据库"
```

而应使用：
```
"为具有商品、用户、订单和库存管理的电商系统设计数据库"
```

### 2. 提供上下文

包含相关信息：
- 目标受众
- 规模要求
- 技术偏好
- 约束或要求

### 3. 迭代优化

从初始输出开始并持续优化：
```
"更新数据库设计以包含评分和评论系统"
"在认证流程中添加社交登录"
```

### 4. 组合技能

您可以按顺序请求多个技能：
```
"首先为博客平台创建PRD，然后设计数据库架构"
```

### 5. 使用工作流代理处理复杂项目

对于完整产品，让全栈产品构建者进行协调：
```
"我想构建一个具有团队协作功能的完整项目管理工具"
```

## 常见工作流

### 工作流1：新产品开发

1. **需求** → 产品经理
   ```
   "为任务管理应用创建PRD"
   ```

2. **数据库** → 数据库设计器
   ```
   "基于上面的PRD设计数据库"
   ```

3. **UI设计** → UI设计师
   ```
   "为任务管理应用设计UI"
   ```

4. **架构** → 解决方案架构师
   ```
   "为该系统创建技术架构"
   ```

### 工作流2：功能添加

1. **需求** → 产品经理
   ```
   "分析添加实时协作功能的需求"
   ```

2. **数据库** → 数据库设计器
   ```
   "更新数据库架构以支持实时协作"
   ```

3. **架构** → 解决方案架构师
   ```
   "为实时功能设计技术解决方案"
   ```

### 工作流3：文档编写

1. **技术文档** → 学术写作
   ```
   "为认证系统创建技术文档"
   ```

2. **API文档** → 文档生成器
   ```
   "为REST端点生成API文档"
   ```

## 技巧和窍门

### 在技能间切换

技能会根据您的请求自动激活。您有三种方式激活特定技能:

**方法一:自然语言(自动检测)**
```
"为电商系统设计数据库"
```

**方法二:显式提及技能**
```
"使用数据库设计器技能为...创建架构"
```

**方法三:斜杠命令(推荐,精确)**
```
/database-designer "电商系统"
```

### 斜杠命令最佳实践

1. **参数要具体**
   ```
   好: /database-designer "包含用户、文章、评论、标签的博客系统"
   差: /database-designer "博客"
   ```

2. **多词参数使用引号**
   ```
   /spw-prd "具有团队协作功能的任务管理应用"
   ```

3. **与自然语言后续结合**
   ```
   /database-designer "电商系统"
   [查看输出后] "在数据库中添加心愿单功能"
   ```

4. **选择正确的命令变体**
   - 单个插件: `/database-designer` (如果安装了单个插件)
   - 套件插件: `/spw-db` (如果安装了 product-development-suite)
   - 两者功能相同,使用您已安装的即可

### 保存输出

所有生成的文件都保存到您的项目目录：
- 数据库设计：`./database/`
- 设计规范：`./design/`
- 文档：`./docs/`
- 架构：`./architecture/`

### 版本控制

生成的文件都是纯文本（Markdown、SQL、JSON），非常适合Git：

```bash
git add docs/database-design.md
git commit -m "添加初始数据库设计"
```

### 团队协作

与您的团队分享生成的文档：
- PRD用于产品讨论
- 数据库架构用于后端开发人员
- UI规范用于前端开发人员
- 架构文档用于基础设施团队

## 故障排除

### 技能未激活

如果激活了错误的技能或没有技能激活：

1. 使请求更具体
2. 使用明确的技能名称
3. 检查插件是否已安装

### 输出不符合预期

如果输出不是您期望的：

1. 提供更多上下文和细节
2. 通过优化请求进行迭代
3. 使用示例来指导输出

### 需要多个技能

对于需要多个技能的复杂请求：

1. 使用工作流代理进行自动协调
2. 分解为顺序请求
3. 在新请求中引用先前的输出

## 高级用法

### 链式请求

通过链式请求构建复杂输出：

```
1. "为聊天应用创建PRD"
2. "为100万+用户设计可扩展数据库"
3. "设计具有WebSocket支持的架构"

```

### 优化循环

迭代至完美：

```
1. "为用户管理设计数据库"
2. "添加OAuth和社交登录支持"
3. "包含所有用户操作的审计日志"

```

### 模板创建

创建可重用的模式：

```
"为多租户SaaS应用创建数据库设计模板"
```
