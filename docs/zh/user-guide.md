# 用户指南

## 概述

本指南将帮助您有效使用单人工作流插件，简化您的产品开发流程。无论您是构建完整产品还是处理特定组件，这些插件都能提供专业化的帮助。

## 快速参考

| 任务 | 技能/代理 | 触发示例 |
|------|-----------|----------|
| 数据库设计 | 数据库设计器 | "为...设计数据库" |
| 产品需求 | 产品经理 | "为...创建PRD" |
| UI/UX设计 | UI设计师 | "为...设计UI" |
| 技术栈选择 | 解决方案架构师 | "帮我选择技术栈..." |
| 学术写作 | 学术写作 | "帮我写研究报告..." |
| 完整产品 | 全栈产品构建者 | "我想构建一个完整产品..." |

## 独立技能

### 1. 学术写作风格

**用途：** 为中英文大学作业提供专业学术写作。

**何时使用：**
- 撰写研究论文
- 创建技术报告
- 准备项目文档
- 起草案例研究和分析

**触发示例：**
```
"帮我写一篇关于云计算的技术报告"
"我需要创建一篇关于AI伦理的研究论文"
"撰写关于微服务架构的学术分析"
```

**输出内容：**
- 自然流畅的文字
- 恰当的学术结构
- 清晰的章节划分
- 专业且易懂的语言

### 2. 数据库设计器

**用途：** 完整的数据库架构设计，包含ER图和SQL脚本。

**何时使用：**
- 设计新数据库架构
- 为应用程序创建数据模型
- 生成ER图
- 编写迁移脚本

**触发示例：**
```
"为电商系统设计数据库"
"为项目管理工具创建架构"
"我需要为社交媒体平台设计数据库"
```

**输出内容：**
- 详细的Markdown设计文档
- 可执行的SQL脚本（MySQL、PostgreSQL、SQL Server）
- DrawDB兼容的JSON/DBML文件
- ER图规范

**特性：**
- 逻辑关系（不包含物理外键）
- 合理的字段大小
- 优化的索引
- 所有对象的强制性注释

### 3. 产品经理

**用途：** 将想法转化为专业的产品需求文档（PRD）。

**何时使用：**
- 分析产品需求
- 创建用户画像
- 编写PRD
- 规划MVP功能
- 定义用户故事

**触发示例：**
```
"为移动任务管理应用创建PRD"
"分析健身追踪平台的需求"
"帮我为在线教育系统定义用户画像"
```

**输出内容：**
- 全面的需求分析
- 用户画像和场景
- 功能规格说明
- PRD文档
- 可行性评估

### 4. UI设计师

**用途：** 将产品需求转化为专业的UI/UX设计规范。

**何时使用：**
- 规划页面结构和信息架构
- 设计UI布局和组件
- 创建色彩方案和视觉指南
- 编写详细的设计规范
- 创建用户流程图

**触发示例：**
```
"为我的仪表板应用设计UI"
"为移动端结账流程创建设计规范"
"规划管理面板的信息架构"
```

**输出内容：**
- 页面结构和导航
- 组件设计规范
- 色彩调色板和排版
- 详细的设计文档
- 用户流程图

### 5. 解决方案架构师

**用途：** 将产品需求转化为可执行的技术架构设计。

**何时使用：**
- 设计系统架构
- 选择技术栈
- 规划部署策略
- 审查架构决策
- 创建技术方案

**触发示例：**
```
"为实时聊天应用设计技术架构"
"帮我为SaaS平台选择技术栈"
"为微服务系统创建部署计划"
```

**输出内容：**
- 完整的架构设计
- 技术栈推荐
- 部署策略
- 基础设施规划
- 架构审查报告

## 工作流代理

### 1. 全栈产品构建者

**用途：** 编排从概念到部署的整个产品开发过程。

**何时使用：**
- 从零开始构建完整产品
- 需要端到端工作流协调
- 希望自动任务编排

**触发示例：**
```
"我想构建一个完整的电商平台"
```

**工作流：**
1. 需求收集 → 产品经理
2. 数据库设计 → 数据库设计器
3. UI/UX设计 → UI设计师
4. 技术架构 → 解决方案架构师
5. 文档编写 → 学术写作
6. 实施协调

### 2. 产品需求分析师

**用途：** 深度分析产品需求和可行性。

**何时使用：**
- 评估新产品想法
- 分析市场适配性
- 评估技术可行性

### 3. 数据库架构规划师

**用途：** 全面的数据库规划和优化。

**何时使用：**
- 复杂数据库设计项目
- 需要性能优化
- 考虑扩展性

### 4. UI/UX设计协调员

**用途：** 协调完整的UI/UX设计流程。

**何时使用：**
- 大规模设计项目
- 多页面/组件设计
- 创建设计系统

### 5. 技术解决方案设计师

**用途：** 高级技术架构和系统设计。

**何时使用：**
- 复杂系统架构
- 微服务设计
- 云原生应用

### 6. 文档生成器

**用途：** 自动生成全面的项目文档。

**何时使用：**
- 创建技术文档
- API文档
- 用户手册和指南

## 最佳实践

### 1. 具体明确

不要使用：
```
"设计一个数据库"
```

而应使用：
```
"为具有商品、用户、订单和库存管理的电商系统设计数据库"
```

### 2. 提供上下文

包含相关信息：
- 目标受众
- 规模要求
- 技术偏好
- 约束或要求

### 3. 迭代优化

从初始输出开始并持续优化：
```
"更新数据库设计以包含评分和评论系统"
"在认证流程中添加社交登录"
```

### 4. 组合技能

您可以按顺序请求多个技能：
```
"首先为博客平台创建PRD，然后设计数据库架构"
```

### 5. 使用工作流代理处理复杂项目

对于完整产品，让全栈产品构建者进行协调：
```
"我想构建一个具有团队协作功能的完整项目管理工具"
```

## 常见工作流

### 工作流1：新产品开发

1. **需求** → 产品经理
   ```
   "为任务管理应用创建PRD"
   ```

2. **数据库** → 数据库设计器
   ```
   "基于上面的PRD设计数据库"
   ```

3. **UI设计** → UI设计师
   ```
   "为任务管理应用设计UI"
   ```

4. **架构** → 解决方案架构师
   ```
   "为该系统创建技术架构"
   ```

### 工作流2：功能添加

1. **需求** → 产品经理
   ```
   "分析添加实时协作功能的需求"
   ```

2. **数据库** → 数据库设计器
   ```
   "更新数据库架构以支持实时协作"
   ```

3. **架构** → 解决方案架构师
   ```
   "为实时功能设计技术解决方案"
   ```

### 工作流3：文档编写

1. **技术文档** → 学术写作
   ```
   "为认证系统创建技术文档"
   ```

2. **API文档** → 文档生成器
   ```
   "为REST端点生成API文档"
   ```

## 技巧和窍门

### 在技能间切换

技能会根据您的请求自动激活。要显式激活技能：

```
"使用数据库设计器技能为...创建架构"
```

### 保存输出

所有生成的文件都保存到您的项目目录：
- 数据库设计：`./database/`
- 设计规范：`./design/`
- 文档：`./docs/`
- 架构：`./architecture/`

### 版本控制

生成的文件都是纯文本（Markdown、SQL、JSON），非常适合Git：

```bash
git add docs/database-design.md
git commit -m "添加初始数据库设计"
```

### 团队协作

与您的团队分享生成的文档：
- PRD用于产品讨论
- 数据库架构用于后端开发人员
- UI规范用于前端开发人员
- 架构文档用于基础设施团队

## 自定义

### 数据库偏好

指定您偏好的数据库：
```
"为...设计PostgreSQL数据库"
"为...创建MongoDB架构"
```

### 设计风格

指导UI设计师：
```
"为...设计极简风格的UI"
"为...创建现代、多彩的设计"
```

### 架构偏好

指定技术偏好：
```
"使用Node.js和React设计架构"
"创建基于Python的微服务架构"
```

## 故障排除

### 技能未激活

如果激活了错误的技能或没有技能激活：

1. 使请求更具体
2. 使用明确的技能名称
3. 检查插件是否已安装

### 输出不符合预期

如果输出不是您期望的：

1. 提供更多上下文和细节
2. 通过优化请求进行迭代
3. 使用示例来指导输出

### 需要多个技能

对于需要多个技能的复杂请求：

1. 使用工作流代理进行自动协调
2. 分解为顺序请求
3. 在新请求中引用先前的输出

## 示例

### 示例1：电商平台

```
用户："我想构建一个完整的电商平台"

全栈产品构建者激活：
1. 创建全面的PRD
2. 设计包含商品、用户、订单、支付的数据库架构
3. 设计店面、商品页、结账的UI
4. 创建微服务技术架构
5. 生成完整文档
```

### 示例2：博客系统

```
用户："为具有评论和标签的博客平台创建PRD"
→ 产品经理创建详细需求

用户："现在为这个博客系统设计数据库"
→ 数据库设计器创建包含文章、用户、评论、标签的架构

用户："设计管理界面"
→ UI设计师创建管理面板规范
```

### 示例3：API服务

```
用户："为RESTful API服务设计架构"
→ 解决方案架构师创建API架构

用户："创建支持此API的数据库架构"
→ 数据库设计器创建适当的架构

用户："生成API文档"
→ 文档生成器创建全面的API文档
```

## 高级用法

### 链式请求

通过链式请求构建复杂输出：

```
1. "为聊天应用创建PRD"
2. "为100万+用户设计可扩展数据库"
3. "设计具有WebSocket支持的架构"
4. "为移动端和Web端创建UI设计"
```

### 优化循环

迭代至完美：

```
1. "为用户管理设计数据库"
2. "添加OAuth和社交登录支持"
3. "包含所有用户操作的审计日志"
4. "针对读取密集型工作负载进行优化"
```

### 模板创建

创建可重用的模式：

```
"为多租户SaaS应用创建数据库设计模板"
```
